We thank Reviewer D6dM for the careful reading. The manuscript should more clearly separate the generic validity statement from experimental design choices.

The guarantees do not require a fixed batch size across time. The setup uses a constant calibration size \(n\) only for notation. With variable calibration size \(n_t\), the split conformal \(p\)-value at batch \(t\) lies on the grid \(\{1/(n_t+1),\ldots,1\}\) and remains conditionally sub-uniform under within-batch exchangeability. When \(n_t\) is fixed in advance or observable before choosing the threshold, the same product/log-budget proof applies with the actual per-batch grids. Variable \(n_t\) affects efficiency and discreteness: smaller batches coarsen the grid and can make sets larger. It does not invalidate mask validity or restart validity. We will state this explicitly.

The framework is score-agnostic in the same sense as split conformal prediction. For any score \(s_t(x,y)\) that is pre-specified or chosen using only past-observable information, conditional exchangeability gives valid conformal \(p\)-values. Different scores can certainly produce different conformal sets and efficiencies; this is a modeling choice, not a change in the mask-valid guarantee, which controls events of the form \(\{p_t(X_t,Y_t)\le \tau_t\}\). We will clarify this and connect it to the regression and classification scores used in the appendix.

The \(K\) values \(\{1,3,5,7,9\}\) were chosen because \(T=20\) and we wanted a compact sweep from singleton masks, recovering marginal per-batch control, to wider rolling windows approaching the anytime end of the spectrum. Odd values avoid nearly redundant neighboring windows while showing the interpolation. We will explain this rationale in Section 6.

We will also temper the empirical wording: the two datasets illustrate the theory across regression/classification and drift/no-drift settings, rather than establishing exhaustive application coverage.
